# Фабрика гипотез: прототип для задачи 1

Прототип для кейса Норникель AI Science Hack «Фабрика гипотез».

Цель: автоматически находить зоны неопределенности в данных по хвостам обогатительных фабрик и формировать проверяемые гипотезы для снижения потерь элемента 28 / элемента 29.

## Структура

```text
01_Data/       Входные данные и подготовленная real-case сводка.
02_Analysis/   Тесты и проверочные сценарии.
03_Results/    Описание ожидаемых результатов и места для экспортов.
src/           Код Streamlit-приложения и Python-пакет.
Dockerfile        Опциональный запуск через Docker.
requirements.txt  Python-зависимости.
pytest.ini        Настройки запуска тестов.
```

## Запуск
Команды нужно выполнять из корня репозитория:
```bash
python -m pip install -r requirements.txt
python -m streamlit run src/app.py
```

После запуска открыть:

```text
http://localhost:8501
```

Если при запуске появляется `FileNotFoundError` для файла из `01_Data/processed`, проверьте, что в репозитории есть два подготовленных файла:

```text
01_Data/processed/nornickel_real_case_data_summary.json
01_Data/processed/nornickel_real_case_knowledge_base.md
```

Без них приложение запускается, но не может построить реальный кейс.

## Запуск через Docker

Docker не обязателен для работы проекта, но его можно использовать для воспроизводимого запуска:

```bash
docker build -t hypothesis-factory .
docker run --rm -p 8501:8501 hypothesis-factory
```

После запуска открыть:

```text
http://localhost:8501
```

## Проверка

```bash
python -m pytest -q
```

Ожидаемый результат:

```text
9 passed
```

## Что реализовано

- Прием разнородных данных по реальному кейсу.
- Карта изученности по фабрикам, элементам и классам крупности.
- Выявление зон неопределенности.
- Генерация проверяемых гипотез.
- Ранжирование гипотез по интерпретируемым критериям.
- Читаемый граф знаний.
- Экспертная обратная связь.
- Экспорт JSON, CSV и DOCX.
